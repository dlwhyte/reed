AI tools connect to your library

Claude, Cursor, ChatGPT — and any other MCP-aware assistant — can read your saved articles and highlights. Your reading becomes context.

BrowseFellow exposes a narrow, read-only agent API that AI tools pull from over HTTPS. When it's connected, your assistant can ground its answers in what you've actually read — not generic web knowledge, not training data. You ask Claude "what have I been reading about attention spans?" and it answers from your library, citing specific articles and your own highlights.

Claude answering from your own reading, not the open web.

Ask it things like…

  • “What have I read about attention spans?”
  • “Summarize the highlights I made this week.”
  • “Find the article where someone argued against remote work.”
  • “Based on what I've saved lately, what should I read next?”
  • “Pull quotes from my shelf about AI agents, with citations.”

The AI tool decides when to call search_library, get_article, or search_highlights. You just ask normal questions.

How it works

  1. Create an API token in Settings → AI tools. Tokens are named + revocable + scoped strictly to your own library.
  2. Paste the provided config snippet into your AI tool of choice — the snippet is generated with your token pre-filled for each supported client (Claude Code, Cursor, Claude.ai, ChatGPT desktop).
  3. That's it. Your AI assistant now has three read-only tools: search_library, get_article, and search_highlights. It decides when to use them; you just ask normal questions.

Supported clients

Claude Code

Add to ~/.claude/mcp.json. Uses the local npm package (runs as a subprocess). 

{
  "mcpServers": {
    "browsefellow": {
      "command": "npx",
      "args": ["-y", "@browsefellow/mcp"],
      "env": { "BROWSEFELLOW_TOKEN": "bft_..." }
    }
  }
}

Cursor

Settings → MCP → Add new MCP server. Same config as Claude Code.

Claude.ai (web)

Settings → Connectors → Add custom connector. Point at the hosted MCP endpoint (no local install needed): 

URL:   https://browsefellow.com/api/agent
Token: bft_...

ChatGPT (desktop / Connectors)

Same URL + token as Claude.ai. Or use the dedicated BrowseFellow Custom GPT (coming to the GPT Store).

Any other client / your own code

The agent API is plain REST. Documentation + OpenAPI spec at github.com/dlwhyte/reed. Three endpoints: search, get article, search highlights. Auth via Authorization: Bearer bft_....

What gets shared

When an AI tool calls the agent API, it can see:

  • Article titles, URLs, summaries, tags, excerpts
  • Up to 8 KB of article text when fetched by ID
  • Your highlights + notes on those articles

It cannot modify your library, access other users' data, or do anything beyond these three endpoints. Every call is scoped to the token's owner.

Token security

  • Tokens are shown once at creation — copy them immediately.
  • Revoke anytime from Settings → AI tools. Revoked tokens stop working immediately.
  • Create one token per tool so you can revoke individually. Don't share a single token across clients.
  • Tokens are stored as SHA-256 hashes. We never see your raw token after creation.

FAQ

Does this cost extra?

The agent API itself is free to use at any tier. Rate limits apply (120–240 requests/minute per token). Creating an API token doesn't require Plus — your library is yours.

Can I use this with a local model?

Yes — anything that speaks HTTP. The REST API is plain, and the MCP server runs against any MCP-compatible client. Ollama users can wire it up via mcp-bridge-style tools.

Does BrowseFellow see what AI tools do with the data?

No. Once the AI tool fetches your content via the agent API, we don't track what it does with it. What we log: per-token request counts + failed auth events, for abuse detection. See the privacy policy.
