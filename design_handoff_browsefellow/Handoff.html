01 · Principles

Paper first. Local first. Cite everything. Few moods, not many.

These aren't decoration. They're constraints. Paper first means the reader is the product — every other surface defers to it in contrast, density, and weight. Local first means the shelf lives on disk; the agent asks permission to leave it. Cite everything means the agent never asserts without a source. Few moods means three reader themes (light, sepia, dark) and four accents, period. No theme shop.

05 · Components, in priority order

Reader reader.jsx

The product. A single article column (560–680px measure), a sticky toolbar on the right (Star / Chat / Research / Theme / Font), and a side panel that slides in for Chat or Research. The panel never covers the article — the article reflows.

  • Body text is clamp(16px, 1.15vw, 20px) by default, tweakable 14–22.
  • Dropcap is Source Serif, terracotta, 4.6em, line-height 0.85. First paragraph only.
  • Blockquotes get a 2px terracotta left-rule and italic Source Serif. Not generic pull-quotes.
  • The TL;DR card is REED.butter + '66' (40% butter) with a serif italic inside. Dismissible in future iterations.

Library library.jsx

Three densities: card, list, mixed. Mixed is the default — big cover tile for the newest, list for the rest. Density is a Tweak because it's a genuine preference disagreement.

Chat + Research panels panels.jsx

Both use Composer, both stream, but they're visually distinct: plum for Chat (conversational), olive for Research (industrial). The trace in Research is collapsed by default — the user is supposed to feel the answer, not the machinery.

Agent trace flows.jsx

Three verbosity modes live in TraceVariations: Compact, Expanded (default), Thinking. The chosen mode is a user setting.

06 · Agent loop

The prototype's Research panel and the RealAgentLoop artboard both call window.claude.complete() with a fallback for when the call fails. In the real app, this is where you wire Cohere:

async function runAgent(query) {
  const turns = [];
  while (turns.length < MAX_TURNS) {
    const response = await cohere.chat({
      model: 'command-r-plus',
      preamble: PREAMBLE,
      tools: [search_library, read_article, search_web],
      chat_history: turns,
      message: query,
    });
    if (response.tool_calls) {
      for (const tc of response.tool_calls) {
        const result = await executeTool(tc);
        turns.push({ role: 'TOOL', tool_results: [{ call: tc, outputs: result }] });
      }
      continue;
    }
    return response; // final answer with citations
  }
}

Three tools ship at launch: search_library (SQLite FTS5 + optional local embeddings), read_article (pulls full text from the shelf), search_web (Tavily). All tool calls and results are rendered into the user-visible trace. No invisible tool use.

08 · States engineers sometimes forget

  • First run. Empty shelf should feel like a welcome, not a void — see EmptyShelf. Three onboarding paths (extension, paste URL, Pocket import) and three sample pieces.
  • Offline. Reader works on cache. Chat + Research go to a gentle "the agent is sleeping" state. Do not block the reader behind a full-screen error.
  • Tool failure. Inline in the trace, with three recovery buttons: Retry, Update key, Skip & answer without web. See ErrorTrace.
  • Long save. The extension popup shows the extraction progress, then the tag review. Don't close automatically — the user wants to confirm the tags.
  • Auto-tag disagreement. Tags are proposed, not imposed. Users can reject; we remember corrections.

09 · Open questions for the team

  1. Chat history. Per-article only, or cross-article? Current prototype is per-article (the chat forgets on close).
  2. Embeddings. Local (nomic-embed-v2 via ONNX) or server-side? Local is in the wireframe but I haven't priced it against model size.
  3. Highlights roll-up. Is the "All highlights" view a timeline or a per-piece clustering? Prototype shows timeline.
  4. Research trace by default. Currently collapsed. Some users will want it expanded by default; should we detect (power user → expanded) or leave it in settings?
  5. Import. Pocket .json is in the empty-state CTA but not built. Priority?
